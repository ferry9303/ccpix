# ccpix

**English** | [中文](README.zh-CN.md)

View the images from your **Claude Code** session transcripts — right in the terminal.

`cc` (Claude Code) + `pix` (pictures). Claude Code stores every image you
paste/upload (and images it reads or generates) as base64 **inside the session
transcript**:

```
<claude-config>/projects/<project>/<session-id>.jsonl
```

The on-disk image cache (`~/.claude/image-cache/<id>/`) is **ephemeral** — Claude
prunes it, so old images disappear. The transcript keeps them all. `ccpix` pulls
the images straight out of the transcript and shows them inline, so you can flip
back through any past conversation's pictures.

## Install

```sh
brew tap ferry9303/ccpix https://github.com/ferry9303/ccpix
brew install ccpix
```

Latest from `main` without waiting for a release:

```sh
brew install --HEAD ferry9303/ccpix/ccpix
```

`chafa` (recommended dependency) is what makes inline images work outside iTerm2.

### Without Homebrew

It's a single dependency-free Python 3 script:

```sh
curl -fsSL https://raw.githubusercontent.com/ferry9303/ccpix/main/ccpix \
  -o ~/.local/bin/ccpix && chmod +x ~/.local/bin/ccpix
```

(or clone and run `./install.sh`).

## Usage

```sh
ccpix                  # list sessions that contain images (id · count · first message)
ccpix 9f3a             # view every image from the session whose id starts 9f3a
ccpix 9f3a -o          # open them in the OS image viewer (Preview on macOS)
ccpix 9f3a -w 60%      # set width (imgcat backend only; e.g. 60% / 600px / auto)
ccpix 9f3a -b chafa    # force a display backend
```

The session id is a prefix — a few characters is enough. Run `ccpix` with no
arguments to see the ids:

```text
$ ccpix
9f3a1c20-...   12 imgs  redesign the onboarding flow
2b7e44a1-...    3 imgs  why does the build fail on CI
0c9d8f55-...    1 img   crop this logo to a square
```

Extracted images are written to `$TMPDIR/ccpix-<id>/` (numbered in order,
de-duplicated), so you can `cp` them out if you want to keep them.

### Try it without any real data

The repo ships a synthetic demo config (generated by `examples/make-demo.py` —
images are drawn on the fly, nothing real):

```sh
CLAUDE_CONFIG_DIR=examples/demo-claude ccpix              # lists one fake session, 3 imgs
CLAUDE_CONFIG_DIR=examples/demo-claude ccpix 0a1b         # shows the 3 generated images
```

## Display backends

`ccpix` auto-detects how to draw images in your terminal:

| Backend  | When                                    | Protocol                     |
| -------- | --------------------------------------- | ---------------------------- |
| `imgcat` | inside iTerm2 (and `imgcat` is present) | iTerm2 inline image          |
| `chafa`  | anywhere else `chafa` is installed      | auto: Kitty / Sixel / iTerm2 |
| `kitten` | `kitten` (kitty) is installed           | Kitty graphics               |
| `open`   | nothing else available                  | OS image viewer              |

`chafa` is the most portable — it detects the terminal and picks the right
graphics protocol itself (Kitty graphics in Ghostty/kitty, Sixel elsewhere,
falling back to colored character blocks). Override auto-detection with `-b` or
`CCPIX_BACKEND=chafa`.

## How it works

- Reads `$CLAUDE_CONFIG_DIR/projects/*/*.jsonl` (defaults to `~/.claude`).
- Recursively scans each transcript line for `{"type":"image","source":{"type":"base64",…}}`
  — covering images you pasted/uploaded **and** images inside tool results
  (e.g. files Claude `Read`).
- De-duplicates by content hash (the same image often appears many times across
  the conversation context) and decodes to real files.

## Requirements

- **Python 3.9+** (standard library only — no pip packages)
- **macOS or Linux** (`open` falls back to `xdg-open`; Windows uses `start`)
- A graphics-capable terminal for inline display (iTerm2, Kitty, Ghostty,
  WezTerm, or any Sixel terminal). Otherwise use `-o` to open in a viewer.

## Release (maintainer)

1. Tag and push:
   ```sh
   git tag v0.1.0 && git push origin v0.1.0
   ```
2. Get the tarball sha256 and paste it into `Formula/ccpix.rb`:
   ```sh
   curl -fsSL https://github.com/ferry9303/ccpix/archive/refs/tags/v0.1.0.tar.gz | shasum -a 256
   ```
3. Commit the formula. `brew install ferry9303/ccpix/ccpix` now installs the stable build.
